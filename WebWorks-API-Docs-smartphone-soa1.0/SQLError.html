Errors in the asynchronous database API are reported using callbacks that have a SQLError object as one of their arguments.

SQLError.UNKNOWN_ERR Number = 0 The transaction failed for reasons unrelated to the database itself and not covered by any other error code.  - BlackBerry OS 5.0+

SQLError.DATABASE_ERR Number = 1 The statement failed for database reasons not covered by any other error code.  - BlackBerry OS 5.0+
 - BlackBerry PlayBook
SQLError.VERSION_ERR Number = 2 The operation failed because the actual database version was not what it should be.  - BlackBerry OS 5.0+
 - BlackBerry PlayBook
SQLError.TOO_LARGE_ERR Number = 3 The statement failed because the data returned from the database was too large. The SQL "LIMIT" modifier might be useful to reduce the size of the result set.  - BlackBerry OS 5.0+
 - BlackBerry PlayBook
SQLError.QUOTA_ERR Number = 4 The statement failed because there was not enough remaining storage space, or the storage quota was reached and the user declined to give more space to the database.  - BlackBerry OS 5.0+
 - BlackBerry PlayBook
SQLError.SYNTAX_ERR Number = 5 The statement failed because of a syntax error, or the number of arguments did not match the number of ? placeholders in the statement, or the statement tried to use a statement that is not allowed, such as BEGIN, COMMIT, or ROLLBACK, or the statement tried to use a verb that could modify the database but the transaction was read-only.  - BlackBerry OS 5.0+
 - BlackBerry PlayBook
SQLError.CONSTRAINT_ERR Number = 6 An INSERT, UPDATE, or REPLACE statement failed due to a constraint failure. For example, because a row was being inserted and the value given for the primary key column duplicated the value of an existing row.  - BlackBerry OS 5.0+
 - BlackBerry PlayBook
SQLError.TIMEOUT_ERR Number = 7 A lock for the transaction could not be obtained in a reasonable time.  - BlackBerry OS 5.0+
 - BlackBerry PlayBook
